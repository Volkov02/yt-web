# yt-web

Простой веб-интерфейс для [yt-dlp](https://github.com/yt-dlp/yt-dlp) на Flask. Позволяет скачивать видео и аудио с YouTube и других сайтов через браузер, не трогая командную строку.

## Возможности

- Скачивание видео в выбранном качестве
- Извлечение аудио (MP3)
- Прогресс-бар в реальном времени (через Server-Sent Events)
- Упаковка в исполняемый файл через PyInstaller (`build.bat` / `build.sh`)
- ffmpeg подтягивается автоматически через `imageio-ffmpeg`

Работает на Windows, Linux и macOS — код кроссплатформенный, платформенная логика (открытие папки, поиск ffmpeg) выбирается автоматически.

## Качество, куки и HD

Чтобы стабильно получать HD (1080p, 4K), YouTube сейчас требует три вещи —
приложение закрывает все три:

1. **Куки авторизации.** В интерфейсе есть выбор источника куков (по умолчанию
   Firefox); приложение берёт их через `cookiesfrombrowser` из профиля браузера,
   где ты залогинен в YouTube. Браузер лучше закрыть перед скачиванием
   (особенно Chrome/Chromium — иначе база куков заблокирована).
2. **JS-рантайм Deno.** YouTube защищает ссылки на форматы JS-задачей («n-challenge»),
   которую yt-dlp умеет решать только через **Deno** (Node для этого не подходит —
   yt-dlp помечает его как `unavailable`). `run.sh` сам ставит локальную копию Deno
   в папку `.deno/` при первом запуске — вручную ничего ставить не нужно.
3. **Скрипт-солвер EJS.** Включён опцией `remote_components=['ejs:github']` —
   yt-dlp подтягивает его сам.

Плюс приложение использует YouTube-клиент **`tv`**: он отдаёт полный набор DASH-форматов
(до 4K) и работает с куками, в отличие от `web` (SABR — только картинки) и
`android`/`ios` (отбрасываются, когда заданы куки).

Без этого yt-dlp сваливается на progressive-формат 18 (360p) — поэтому раньше
длинное видео весило подозрительно мало.

## Установка

Требуется **Python 3.9+**.

```bash
pip install -r requirements.txt
python app.py
```

Откроется браузер на `http://127.0.0.1:5757`.

## Запуск

**Windows:**

```cmd
run.bat
```

**Linux / macOS:**

```bash
./run.sh
```

`run.sh` сам создаёт виртуальное окружение `.venv`, ставит зависимости и запускает сервер.

## Сборка в исполняемый файл

**Windows** — результат `dist/YT Downloader.exe`:

```cmd
build.bat
```

**Linux / macOS** — результат `dist/yt-downloader`:

```bash
./build.sh
```

## Зависимости

- `flask>=3.0.0`
- `yt-dlp>=2024.1.0`
- `imageio-ffmpeg>=0.4.9`
